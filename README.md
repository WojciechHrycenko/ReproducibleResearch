## Project Overview

This repository contains the final project for the **Reproducible Research** course. The primary focus of this project is the process of reproduction, documenting challenges, implementing good coding practices, and collaborating effectively via Git, rather than analytic correctness.

The specific topic will be selected from the course materials and confirmed with the coordinator.

## Assessment of the Course: Project Details

**Grading Structure (Total: 50 pts)**
* Final Project: **40 pts**
* Presentation: **10 pts**

### 1. Project Must-Haves (40 pts)
* **Repository & Collaboration (15 pts):** Project stored in a repository from the start. Must include a viewable history of well-described contributions from all collaborating team members.
* **Reproducibility & Documentation (15 pts):** Code and results provided with appropriate documentation (e.g., Markdown) to ensure full reproducibility, including details like software versions.
* **Code Quality (10 pts):** Code structured in a clean and user-friendly format.

### 2. Core Evaluation Criteria
The project is not aimed at testing programming, econometric, or statistical skills. The correctness of analytical reasoning is not graded. Instead, the focus is on:
* **The process of reproduction:** Did it succeed? What were the challenges? What was missing in the original source? What didn't work? What could have been done differently in the original work?
* Making the team's own analytical process fully reproducible.
* Quality of documentation and good coding practices.
* Effective collaboration and version control via Git.

### 3. Presentation (10 pts)
* Scheduled during the last 2-3 meetings of the course.
* Duration: Approximately **20 minutes**.
* Requirements: 
  * All team members must actively participate.
  * Briefly cover what the project is about, how it was done, and how it works.
  * Include a short presentation of the code.
  * The project will be run live from a lab computer after pulling directly from this GitHub repository.
